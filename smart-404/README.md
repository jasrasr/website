# jasr.me Smart 404

These files live in the public `website` repo under `smart-404/`. After GitHub changes sync to Hostinger, move the deployable files from Hostinger's `/github/smart-404/` checkout to the site's web root.

Do not upload these files directly from the local computer for the normal workflow.

## Deployable files

- `.htaccess` updates YOURLS routing so exact `/github/<request>` matches are tried before YOURLS.
- `smart-404.php` logs misses, checks `data/smart-404-map.json`, then tries conservative fuzzy matching against `/github`.
- `smart-404-admin.php` provides the password-gated admin page for logs, mappings, and malicious patterns.
- `data/.htaccess` blocks direct browser access to logs and mappings.
- `yourls-plugin/user/plugins/smart-404-jasrasr/plugin.php` routes unknown YOURLS keywords to `smart-404.php`.

## Server-managed files

These live files are ignored in Git and should not be overwritten from the repo:

- `data/404-requests.jsonl` stores one JSON object per missed URL.
- `data/smart-404-admin-password.php` stores the real admin password hash.
- `data/smart-404-map.json` stores manual mappings.
- `data/smart-404-malicious.json` stores exact, prefix, and contains patterns that route to a separate logged 404 page.

Sample files are committed for reference:

- `data/404-requests.sample.jsonl`
- `data/smart-404-admin-password.sample.php`
- `data/smart-404-map.sample.json`
- `data/smart-404-malicious.sample.json`

Keep redirects as 302 until the mappings have been tested.

## YOURLS unknown keyword plugin

Move `yourls-plugin/user/plugins/smart-404-jasrasr/plugin.php` into the YOURLS install on Hostinger, then activate **Smart 404 Unknown Keyword** in the YOURLS admin plugin page. This is needed because YOURLS handles unknown shortlink keywords before Apache `ErrorDocument 404` runs.

## Admin setup

Before using `smart-404-admin.php`, create `data/smart-404-admin-password.php` on the server. Start from `data/smart-404-admin-password.sample.php`, but replace the sample hash with a real password hash generated by PHP `password_hash()`.

The admin page can add manual mappings from either the manual form or a log row. When saving a mapping, use a site-local target such as `/github/mpg/`; full external URLs are not redirected by `smart-404.php`. The save form can also remove existing log entries whose normalized path matches the mapping source, or leave them in place and show how many matching entries remain.

The admin page can also mark log rows as malicious patterns. Pattern types are:

- `exact` matches only the normalized path.
- `prefix` matches that path and anything below it.
- `contains` matches the value anywhere in the normalized path.

Malicious matches are checked before manual maps and fuzzy GitHub matching, then rendered as a separate 404 page that says the request has been logged.
